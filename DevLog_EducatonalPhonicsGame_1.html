Educaional Phonics Card Game Dev Log 1 : The Concept

09/02/2026

Analizing Educational Games:

Videogames are complicated, and they need a way of teaching how to play to players. Mechanics have been taught several ways throughout the years: from manuals to tutorials to tool tips. There are so many methods to teach through games, but much of the learning that happens in games is purely subconscious. After all humans are well adapted to pattern recognition and as such many player will figure out minor things about games just through interacting with the game world, from how many shots it takes to kill an enemy, to what jumps you can make to enemy attacks and how to avoid them. This is all taught to players via repetition. If a player interacts with a system or gameplay loop for long enough, they will start to understand. Gameplay can be designed in such a way to subtly force players to interact with the game systems in a specific way, so that players will notice and make these interactions intentionally in the future. See the AntLion grub in HL2:E2 Developer comentary.

Challenges of making educational games:

Games are filled with rules. These rules, when understood by players, allow players to make and execute plans. Some educational concepts (like mathematics) follow these rules strictly, making them perfect candidates for educational games. Unfortunately, the English language is a hodgepodge of several language roots, and the rules of English have several exceptions. This makes phonics a difficult subject to turn into a game.

Types of educational games:

Educational automation:

Educational automation games are simply automated versions of traditional educational tools such as worksheets memory games etc. Games are often not engaging to players. Despite being the simplest of the 3 types of educational games it does have some benefits over traditional teaching methods as it allows for immediate feedback.

Games with educational Paint:

Some educational games are simply a game with an educational coat of paint on it. This form of game is much more engaging when compared to educational automation games, however the educational aspects of it are an afterthought and are often poorly incorporated into game play. The educational material is separate from the core mechanics of the game, unlike in the next form of game.

Games that happen to be educational:

the gold standard for educational games are games whose mechanics happen to be academically relevant, unfortunately this kind of educational game is often limited to STEM Fields. Some common examples of this form of game are: Kerbal space program (teaches rocket science, orbital mechanics), Sim City (teaches fanatical and city planning), bridge builder (teaches structural engineering and physics) Balatro (teaches probability and order of operations). This is ideal for educational games as the game is engaging outside of an educational context. The educational elements are inseparable from the gameplay.

Game Structures

Linear:

Linear sees players move through a series of bespoke levels one after another. In the context of an educational experience, each level can act as a lesson. However, linear experiences fall flat for educational programs as each level only needs to be completed once and as such struggles to ensure a robust understanding of content material.

Cyclic:

cyclic game structures involve players repeatedly replaying the same level over and over again to achieve mastery of that specific level. This format is a much better fit for education when compared to linear experiences as repeated playthroughs help familiarizing players with the mechanics through repetition. However the format is drawn back by the incentive to players replaying levels is purely up to the player, this means if a player is not thoroughly engaged by the program (which is often the case when dealing with educational games) the player has no reason to play the levels more than once. The format is also prone to memorization/ optimization: this means that it is likely that players will memorize the most optimal path through a level, prioritizing memorization over mastery of the mechanics. Linear:

Rogue style games are a format where players are given a task, they will attempt a “run” to reach this goal and acquire upgrades along the way. Upon failing the run all abilities and progress towards the goal is lost. Each run is subjected to randomness that makes each run distinct. The abilities / upgrades often incentivize prioritization of differing playstyles, this incentivizes mastery offer all aspect of the mechanic, rather than focusing on one strategy. Rogue style of games would be the best format to use in an educational experience.
The rogue style game also features several pros for an educational experience:
-As no upgrades and abilities are carried over from one run to another, the need for progression tracking/ saving/ accounts is not needed to be implemented.
Each run is designed to last 10-20 min so several run attempts can be done in a computer lab session. As well as a reasonable time spent at home learning.
Randomized runs incentivize experimentation with various parts of the content material.

Designing for the target audience:

When designing a game, it is important to design the demographic. For instance, the age demographic of LEM Phonics (children aged 4-10) has grown up in the digital world with devices constantly fighting for their attention. From coco Mellon and dancing fruit videos to the short form video formats of TikTok and YouTube shorts, games that are often played by Gen Alpha include: Fortnite, ROBLOX, Mario games and Minecraft as evidenced by this Statistic.
As the game will need to be designed for such a young audience with differing technical and game literacy, both from students and educators.
Educational experiences should be equally engaging to a young audience as one of these other games. But how do we do this? The key to making a topic like phonics more engaging is less in the gameplay but rather by making interactions satisfying, of what game designers refer to as “Juice” . Juice in a game describes the process of adding supplementary animations/ effects to actions in order to make them more engaging.

The design complexity of the game:

As the game is designed to target several age groups with wildly differing understandings of the material and technical literacy, the game will need several levels of difficulty in both game play and different interaction complexity.

Game Pitch

The final project should have several game modes to accommodate several levels of digital literacy and understanding of Phonics.

The Card Design:

the card design features the Phonic letters displayed predominantly oh the front of a square card. A line of pips going down vertically along the left edge of the card. The quantity of pips, is derived by how many sounds the phonic has. The highlighted pip is displays which sound is currently selected.

variables of the card:

-String: Phonic Letters (the letters displayed on the card)
-AudioClip[]: Sonds (the audio clips stored in an array)
-String[]: Phonetic Sound (either a phonetic spelling or the phonetic alphabet version of the sound)
-String[]: Exemplar Words (an example word that the sound appears in)(optional)
-Material: Phonic Class Material (single phonograms multiple phonograms, ect all have a different card backing to help distinguish between them)

The Card Drawer

In some experiences (card viewer, word maker) the player will have access to all the cards in the game. To achieve this we will use the “card drawer” a pop up menu that allows players to scroll through all the cards, once a card is selected the card drawer will collapse allowing players to add it to the scene.

The Experiences:

Card viewer:

The simplest of the experiences featuring a collection of cards with each of the phonograms, players can select a card and move it onto a card slot. This card slot has controls along the side for changing the selected sound and playing the sound.
This is intended to teach the phonics and can be used in guided learning or solo experimentation.

Card Test:

A sound is played and players need to select the card (out of 3 –5 cards) that plays that sound.

Word Maker:

Players add any cards onto card slots to spell words. A button on the side will play each audio clip so the game will say the word. (to avoid the system spelling bad words the final string will be compared to a “badWords.txt” file. If a word from this list is detected the game will return the cards to the card drawer)

The Game (as yet unnamed)

A rogue style mode where players play words using phonics in order to score points to reach a threshold point. If a player is unable to reach the threshold score in a round the run is over and the player will need to start a new run. to make the game more interesting, this experience will feature modifier cards that will alter the score. Modifier cards could, for instance, give double points if the word contains a Successive Seventeen Phonogram. These modifier cards incentivize players to modify how they play in order to increase the score.
This experience is greatly influenced by Balatro and it will act as the general framework for this experience.
